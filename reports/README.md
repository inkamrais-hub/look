# AAN — Algebraic Attention Network

## 项目定位

AAN 是一个纯代数运算路线的序列模型研究项目。与主流 Transformer 架构不同，AAN **彻底摒弃**了 softmax 注意力、LayerNorm、位置编码等标准组件，全部用**可学习的代数运算**替代。

## 核心理念

- **离散状态空间**：将连续向量空间映射到有限个离散状态，用 Cayley 表（代数运算表）建模状态间的交互
- **代数归约**：用迭代的代数运算聚合序列信息，替代自注意力机制
- **无注意力机制**：完全不使用 QKV 注意力，也没有 softmax
- **无归一化层**：不使用 LayerNorm / BatchNorm
- **无位置编码**：序列位置信息通过代数交互顺序隐式编码

## 架构要点

```
输入 → 状态嵌入 → [代数交互 × 归约 × 恢复]^L → 状态解码 → 输出
```

每一层中：
1. 连续向量被编码为离散状态分布
2. 多个代数运算表在邻域位置间执行状态交互
3. 全局归约将序列聚合为单一状态
4. 离散状态被解码回连续空间
5. 代数残差连接保留信息流

## 验证结果

| 指标 | AAN (v1) | Transformer | 对比 |
|------|----------|-------------|------|
| 参数量 | 61K | 421K | **7× 更少** |
| 最佳 PPL | 12.65 | 9.19 | 差距 27% |
| 吞吐量 | 122K tok/s | 145K tok/s | 接近 |
| 峰值显存 | 35 MB | 66 MB | **节省 46%** |
| 参数效率 | 0.207 PPL/1K | 0.022 PPL/1K | **9.5× 更优** |

AAN 用 **1/7 的参数量**达到了接近 Transformer 的困惑度，同时具有更低的显存占用和可比的生成质量。

## 技术栈

- **框架**：PyTorch
- **核心运算**：Gumbel-Softmax 离散化、可学习代数表、迭代归约
- **评估**：困惑度、吞吐量、生成质量（熵/多样性/JS 散度）
- **数据**：TinyShakespeare
